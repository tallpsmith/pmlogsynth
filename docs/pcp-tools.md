# PCP Tools Reference

**Version**: PCP 7.1.1
**Location**: `/usr/local/bin/`

Use these tools to inspect and validate archives generated by pmlogsynth.

---

## Standard Validation Workflow

```bash
# 1. Generate a test archive
pmlogsynth -o ./generated-archives/spike spike.yml

# 2. Check archive integrity
pmlogcheck ./generated-archives/spike

# 3. Dump archive metadata (time range, metrics present)
pmdumplog -l ./generated-archives/spike

# 4. Verify specific metric values
pmval -a ./generated-archives/spike kernel.all.cpu.user

# 5. Summarise statistics across entire archive
pmlogsummary ./generated-archives/spike

# 6. View system-level stats in tabular form
pmstat -a ./generated-archives/spike

# 7. Cross-check metric descriptors (type, sem, units)
pminfo -a ./generated-archives/spike -d kernel.all.cpu.user
```

---

## Primary Inspection Tools

| Tool | Purpose | Key flags |
|------|---------|-----------|
| `pmlogcheck` | Validate archive structure and temporal consistency | _(no flags needed for basic check)_ |
| `pmdumplog` | Dump archive contents in human-readable form | `-l` label/time range; `-a` all metrics; `-m` mark records |
| `pmlogdump` | Alias for `pmdumplog` on some PCP builds | same as above |
| `pmlogsummary` | Per-metric min/max/avg/stdev across archive | `-H` headers; `-l` instance labels; `-N` no interpolation |
| `pmval` | Display time-series values for one metric | `-a` archive; `-i` instance filter; `-t` interval |
| `pmstat` | System-level stats table (like `sar`) | `-a` archive; `-t` interval; `-s` samples |
| `pminfo` | Metric descriptors: type, semantics, units, help | `-a` archive; `-d` descriptor; `-t` one-line help; `-T` full help |
| `pmrep` | Flexible columnar reporter, CSV/JSON output | `-a` archive; `-o` output format; `-F` field width |

---

## Transformation Tools

| Tool | Purpose |
|------|---------|
| `pmlogextract` | Extract a time window or metric subset from an archive |
| `pmlogreduce` | Downsample an archive to a coarser interval |
| `pmlogrewrite` | Rename/retype metrics (useful when schema evolves) |
| `pmlogcompress` | Compress `.0`/`.1`/... data volumes in-place |

## Quick-Reference One-Liners

```bash
# List all metrics present in an archive
pmdumplog -a ./generated-archives/spike | grep '^[a-z]' | sort -u

# Show archive time range (start â†’ end)
pmdumplog -l ./generated-archives/spike

# Dump all samples for a metric with timestamps
pmval -a ./generated-archives/spike -f 3 kernel.all.cpu.user

# Dump every metric value (verbose â€” pipe to less)
pmdumplog -a ./generated-archives/spike | less

# CSV export of CPU metrics for plotting
pmrep -a ./generated-archives/spike -o csv kernel.all.cpu.user kernel.all.cpu.sys

# Check a single metric's descriptor (type/sem/units)
pminfo -a ./generated-archives/spike -d disk.all.read

# Extract a 5-minute window into a new archive
pmlogextract ./generated-archives/spike \
    -S '00:01:00' -T '00:06:00' \
    ./generated-archives/spike-window

# Validate archive integrity, exit non-zero on errors
pmlogcheck ./generated-archives/spike && echo "OK"
```

## Notes

- Archive base name only (no extension) for all tools: `./generated-archives/spike` not `./generated-archives/spike.meta`
- `pmval` interpolates by default; use `-r` for raw (uninterpolated) counter values
- Counter metrics (e.g. `disk.all.read`) display as rates; use `-U` to see raw accumulated values
- `pmlogsummary` skips metrics with only one sample â€” generate at least a few seconds of data
