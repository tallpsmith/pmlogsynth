# PyPI Deployment Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate pmlogsynth from setuptools to hatchling + hatch-vcs and add a GitHub Actions release workflow that publishes `pcp-pmlogsynth` to PyPI via OIDC Trusted Publishers.

**Architecture:** Replace the build backend, derive version from git tags at build time (eliminating hardcoded version strings), and add a release workflow triggered by GitHub Release events. The workflow validates tag-version consistency before publishing.

**Tech Stack:** hatchling, hatch-vcs, pypa/gh-action-pypi-publish, GitHub Actions OIDC

**Spec:** `docs/superpowers/specs/2026-03-20-pypi-deployment-design.md`

## Chunk 2: Release Workflow

### Task 6: Create the release workflow

**Files:**
- Create: `.github/workflows/release.yml`

- [ ] **Step 1: Create the workflow file**

```yaml
name: Release to PyPI

on:
  release:
    types: [published]

permissions:
  id-token: write
  contents: read

jobs:
  validate:
    name: "Validate tag matches package version"
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
          fetch-tags: true

      - name: Install tools
        run: python3 -m pip install hatch packaging --break-system-packages

      - name: Validate version
        run: |
          TAG="${GITHUB_REF_NAME}"
          # Strip leading 'v' prefix (hatch-vcs default tag pattern is v*)
          TAG_VERSION="${TAG#v}"
          # Normalize through PEP 440
          TAG_NORMALIZED=$(python3 -c "from packaging.version import Version; print(Version('${TAG_VERSION}'))")
          # Get version from hatch (reads hatch-vcs config)
          HATCH_VERSION=$(hatch version)
          echo "Tag version (normalized): ${TAG_NORMALIZED}"
          echo "Hatch version: ${HATCH_VERSION}"
          if [ "${TAG_NORMALIZED}" != "${HATCH_VERSION}" ]; then
            echo "::error::Version mismatch! Tag=${TAG_NORMALIZED}, Hatch=${HATCH_VERSION}"
            exit 1
          fi
          echo "Versions match."

  pypi:
    name: "Build and publish to PyPI"
    runs-on: ubuntu-latest
    needs: validate
    environment: pypi
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
          fetch-tags: true

      - name: Install build tools
        run: python3 -m pip install build --break-system-packages

      - name: Build sdist and wheel
        run: python3 -m build

      - name: Publish to PyPI
        uses: pypa/gh-action-pypi-publish@release/v1
```

Notes:
- `hatch` (the CLI) is installed in the validate job, not `hatchling` (the library).
  `hatch version` reads the project's hatch-vcs config and returns the derived version.
- The `v` prefix is stripped before PEP 440 normalization — hatch-vcs default tag
  pattern expects `v*` tags and strips the prefix automatically.
- For a TestPyPI dry-run of the first release, temporarily add
  `repository-url: https://test.pypi.org/legacy/` to the publish step and register
  a Trusted Publisher on test.pypi.org instead. Revert after confirming it works.
